## Архитектура системы

### Observer

Интерфейс Observer предназначен для получения событий от Observable.

Содержит следующие методы:

```java
void onNext(T item);
void onError(Throwable t);
void onComplete();
```

Назначение методов:

* onNext() — получение очередного элемента потока;
* onError() — обработка возникшей ошибки;
* onComplete() — уведомление о завершении потока.

### Observable

Класс Observable представляет источник данных и отвечает за генерацию событий.

Создание Observable осуществляется через статический метод:

```java
Observable.create(...)
```

Для получения данных используется метод:

```java
subscribe(...)
```

который позволяет подписать объект Observer на поток событий.

### Disposable

Интерфейс Disposable реализует механизм отмены подписки.

Основные методы:

```java
void dispose();
boolean isDisposed();
```

После вызова метода dispose() подписчик перестаёт получать новые события.

### Scheduler

Scheduler представляет собой абстракцию для управления потоками выполнения.

Интерфейс содержит метод:

```java
void execute(Runnable task);
```

который позволяет запускать задачи в различных типах потоков.

## Использование паттерна Observer

В основе разработанной библиотеки лежит паттерн проектирования «Наблюдатель» (Observer Pattern).

Observable выступает источником событий, а Observer является подписчиком этих событий.

При возникновении нового элемента Observable уведомляет всех подписчиков через метод onNext(). В случае возникновения ошибки вызывается метод onError(), а после завершения потока — метод onComplete().

Данный подход обеспечивает слабую связанность между источником данных и потребителями данных, что является одним из основных принципов реактивного программирования.

## Реализованные Scheduler

### IOThreadScheduler

Реализован на основе CachedThreadPool.

Предназначен для:

* работы с файлами;
* сетевых запросов;
* операций ввода-вывода.

### ComputationScheduler

Реализован на основе FixedThreadPool.

Предназначен для:

* вычислительных задач;
* обработки данных;
* CPU-bound операций.

### SingleThreadScheduler

Реализован на основе SingleThreadExecutor.

Предназначен для:

* последовательного выполнения задач;
* обработки событий в одном потоке;
* сценариев, требующих сохранения порядка выполнения.

## Реализованные операторы

### map()

Оператор выполняет преобразование элементов потока.

Пример использования:

```java
.map(x -> x * 10)
```

### filter()

Оператор выполняет фильтрацию элементов потока по заданному условию.

Пример использования:

```java
.filter(x -> x > 5)
```

### flatMap()

Оператор преобразует каждый элемент потока в новый Observable.

Пример использования:

```java
.flatMap(x ->
    Observable.<Integer>create(e -> {
        e.onNext(x * 10);
        e.onComplete();
    }))
```

## Управление потоками выполнения

Для управления потоками были реализованы методы:

### subscribeOn()

Позволяет выполнять подписку на указанном Scheduler.

Пример:

```java
observable.subscribeOn(new IOThreadScheduler());
```

### observeOn()

Позволяет обрабатывать элементы потока в указанном Scheduler.

Пример:

```java
observable.observeOn(new SingleThreadScheduler());
```

## Обработка ошибок

В библиотеке реализована централизованная обработка ошибок.

При возникновении исключения оно передаётся подписчику через метод:

```java
onError(Throwable throwable)
```

Это позволяет избежать аварийного завершения программы и обеспечивает корректную обработку исключительных ситуаций.
